# indie-product-direction

从"我想做什么"转向"市场已经证明了什么"。你不是在找灵感，你是在筛假设。

这是一套完整的独立开发方向决策体系，基于 Flora 公众号系列文章整理，覆盖从"不知道做什么"到"有一个可验证的产品假设"的全流程。

---

## 核心理念

**真需求 = 痛点 × 高频 × 付费**

问三个问题：
1. **用户是否已经在骂？** — 有人抱怨/求替代品吗？
2. **用户有没有花过钱？** — 已有付费方案解决这个问题吗？
3. **问题是不是高频？** — 每天/每周发生，还是每年一次？

> 三个都满足 → 值得做 | 满足两个 → **谨慎，但可以继续分析** | 只满足一个 → 基本放弃

**关键 nuance：** 频率低但单次价值高的场景（如简历优化、简历模板）依然可能是好方向，只是不适合做订阅制，更适合一次性付费或按次付费。不要因为"频率低"就立刻放弃，而是要考虑收费模式是否能匹配频率。

---

## 流程选择：根据用户输入类型走不同路径

用户输入无非三种情况，先判断再选择路径：

```
用户说"不知道做什么，帮我找方向"
  └→ 走路径 A：先做用户诊断 → 再进入信号发现（Phase 1）

用户说"我有个方向，帮我分析值不值得做"
  └→ 走路径 B：先做真需求评估 → 再纵深分析（Phase 3）

用户说"我有好几个想法，帮我选一个"
  └→ 走路径 C：真需求评估 → 五层漏斗（Phase 2）
```

**不要跳过这个判断。** 用户没说清楚时，先问清楚。

---

## 路径 A：用户诊断 + 信号发现

### 第一步：用户诊断（仅适用于"不知道做什么"的用户）

先问 4 个问题缩小范围，而不是直接去 Reddit 大海捞针：

1. **你的技能栈是什么？** — 前端/后端/AI/全栈？熟悉的语言和框架？
2. **你有没有行业/领域知识？** — 跨境/教育/医疗/金融？有 domain knowledge 的方向成功率远高于纯技术方向
3. **你愿意投入多少时间？** — 兼职（每周 10h）/全职？决定项目规模和复杂度
4. **你希望什么样的收费模式？** — 订阅 SaaS / 一次性付费 / 免费+广告？

> 记录答案。**如果用户没有回答全部问题，基于最合理的默认假设继续，但必须明确标注'以下是我的假设，请确认'。** 然后带着这些约束进入信号发现。

### 第二步：四大信号源采集

#### 信号源 A：Reddit — 搜真实抱怨

用户不是在展示，而是在求助。他们已经承认有问题、主动找方案、愿意花时间表达。

**操作：**
1. 根据用户诊断结果定位 subreddit：AI→r/AI, SaaS→r/SaaS, SEO→r/SEO, App→r/AppIdeas, 跨境→r/Dropshipping
2. 搜痛点关键词（非产品名）：`looking for` / `alternative to` / `too expensive` / `hate this` / `manual work` / `frustrated` / `workflow`
3. **看重复不看点赞** — 一个月内反复出现才是需求
4. 重点搜替代品：`alternative to X` / `better than X` / `cheap version`
5. **需求翻译法** — 把抱怨直接翻译成产品定义：

| 用户说 | 翻译后 |
|---|---|
| "截图调整尺寸太烦" | AI Screenshot Generator |
| "客服消息太乱" | Telegram Help Desk |
| "SEO 内容生成太慢" | AI SEO Automation Tool |

#### 信号源 B：Product Hunt — 看已验证产品

- **不要研究榜首**，关注排名 **10～50 名**的中部产品
- **先看定价页**不是首页（定价页暴露商业逻辑）
- 记录：免费版限制了什么、最低付费档、高级版多了什么、有没有企业版
- 积累项目数据库（记录 50 个产品后规律会自然浮现）

#### 信号源 C：Google Trends — 萌芽需求

- **看斜率不看绝对值** — 上涨的小市场（+500%）比稳定大市场（+5%）更值钱
- 不搜大词，搜长尾：`AI screenshot maker` / `Telegram customer support`
- 利用 Related Queries 的 "Breakout" 爆发词（已接近产品名称）
- 关注区域数据：需求不是全球同时爆发的

#### 信号源 D：评论挖掘 — 竞品差评中的需求

用户的差评里写满了你的产品定义。这是最被忽视但最有价值的信号源。

**操作：**
1. 定位评论平台：SaaS→G2/Capterra, App→App Store/Google Play, 电商→Amazon
2. 找同类产品或替代品，收集低分评论（2-3 星最有价值——差评是情绪宣泄，中评是真实吐槽）
3. **关注模式而非个案** — 同一条抱怨出现 5 次以上就是真需求
4. **挖掘模式：**

| 评论信号 | 翻译成机会 |
|---|---|
| "太贵了，有没有更便宜的" | 价格切口 — 做轻量版 |
| "功能太多，我只想要一个简单功能" | 复杂度切口 — 做单点工具 |
| "不适合我们的场景" | 场景切口 — 做垂直版 |
| "XX平台不支持" | 平台切口 — 做跨平台版 |
| "对非英语用户不友好" | 本地化切口 — 做语言版 |
| "学习曲线太陡" | 体验切口 — 做得更简单 |

**目标产出：** 至少 15-20 条真实用户信号（Reddit + PH + GT + 评论挖掘） → 5 个候选方向 → 进入五层漏斗筛选

## 路径 B：真需求评估 + 纵深分析（用户已有具体方向时）

**不要走五层漏斗——五层漏斗是用来从多个候选中筛出一个的，不适合单方向分析。**

### 第一步：真需求评估

```
痛点（1-5） × 高频（1-5） × 付费（1-5） = 总分
```

给每个维度打分：

| 维度 | 1 分 | 5 分 | 怎么判断 |
|---|---|---|---|
| 痛点 | 用户不解决也能凑合 | 不解决就损失钱/客户/时间 | 搜 Reddit/评论看抱怨强度 |
| 高频 | 一年一次或更少 | 每天/每周都用 | 问自己用户多频繁遇到这问题 |
| 付费 | 全是免费工具 | 已有付费产品且用户愿意花钱 | 搜竞品定价页，看有没有人付过钱 |

> **总分 ≥ 8** → 值得深入分析 | **4-7 分** → 谨慎，先做低成本验证 | **< 4 分** → 基本放弃

### 第二步：纵深分析（Phase 3）

直接进入 Phase 3 的竞品分析、拥挤度、付费意愿评估（见下方）。

---

## 路径 C：真需求评估 + 五层漏斗（用户有多个候选方向时）

先对每个候选方向做真需求评估打分，排掉明显不行的，剩下的进入五层漏斗。

### 五层漏斗方向筛选

#### 第一层：快速排除

五个问题，三个"否"即排除：
1. 问题是否具体？ 2. 用户能否感知？ 3. 现有方案是否麻烦？ 4. 用户可能付费？ 5. 能否做出小版本？

#### 第二层：按用户问题分类

不按产品形态分类（App/插件/SaaS），按用户问题。每个拆成三件事：

```
服务谁 → 解决什么问题 → 这个问题有什么代价
```

#### 第三层：寻找重复信号

- 同一群体反复抱怨同一问题？
- 不同产品围绕同一工作流切不同环节？
- 用户已形成大量替代方案？

> 一次→观察区；五次以上反复出现→候选方向

#### 第四层：五维打分

每个方向按五个维度打分（各 1-5 分），**总分最高的方向优先进入验证**：

| 维度 | 1 分 | 5 分 | 评估方法 |
|---|---|---|---|
| 痛点强度 | 可有可无 | 不解决就难受 | 看用户抱怨频次和强度 |
| 付费意愿 | 免费用 | 已在花钱 | 看竞品定价和用户评论 |
| 获客清晰度 | 一片模糊 | 知道去哪找 | 能否列出 3 个用户聚集地 |
| 实现成本 | 耗时半年 | 一周搞定 | 评估 MVP 复杂度 |
| 竞争压力 | 巨头林立 | 没人认真做 | 数竞品数量和质量 |

> 打分的意义不是精确，是停止"凭感觉喜欢"。

#### 第五层：7 天验证原则

把最终选出的方向压缩成：**具体人群 + 具体场景 + 具体问题 + 7 天验证动作**

验证方式（不需要完整产品）：
- 落地页 + 邮箱订阅
- Notion 模板
- 冷邮件
- **先收费再人工交付**（最强验证信号）

## Phase 3：纵深分析（三种路径都用到）

对经过筛选的候选方向（1-3 个），用以下工具深度解剖。

### 竞品分析

#### 三圈竞品扫描

| 圈层 | 类型 | 意义 | 机会判断 |
|---|---|---|---|
| 第一圈 | 直接竞品（同类产品） | 市场已有标准答案 | 看他们哪里做不够 |
| 第二圈 | 间接竞品（ChatGPT/人工服务/大平台子功能） | 暴露真实替代行为 | 看用户为什么选它们而不是同类产品 |
| 第三圈 | **土办法竞品**（Excel/手动流程/微信群） | **真正的机会常藏在这里** | 有人用土办法 = 需求真实但没人做好产品 |

#### 三线拆解

| 拆解线 | 核心问题 | 收集什么信息 |
|---|---|---|
| 获客线 | 用户怎么知道他们的？ | 流量来源、SEO/ASO、社媒渠道、KOL 推广 |
| 转化线 | 访客怎么变成用户的？ | 落地页结构、漏斗步骤、注册门槛 |
| 留存线 | 用户为什么持续付费？ | 核心功能粘性、数据积累、切换成本 |

#### SEO 竞品扫描（量化维度）

如果能获取 SEO 数据，从这些维度量化竞品：

| 维度 | 意义 | 数据来源 |
|---|---|---|
| Domain Authority / DR | 网站整体权重 | SEO 工具（Ahrefs/Moz） |
| 月均流量 | 竞品的需求规模 | SEO 工具 |
| Top 10 关键词数 | 覆盖的关键产品场景 | SEO 工具 |
| 反链总量与质量 | 内容/PR 投入力度 | SEO 工具 |
| 主要流量来源（搜索/直接/外链） | 获客模式 | Similarweb |

**快速判断：** DA 高、流量大但评论口碑不佳 → 有机会用更好的产品体验切走；DA 低但增长快 → 正在被市场验证的新方向。

#### GEO/AI 引用分析

大语言模型（ChatGPT/Perplexity）开始成为用户的"搜索引擎"。检查：

- **竞品内容是否被 AI 引用？** — 搜 `competitive analysis` / `how to do X` + 竞品名，看 AI 回答中是否引用其内容
- **被引用的内容类型：** 定义性文章、统计数据、清单类内容最容易被引用
- **缺口：** 如果竞品不被引用，你就有机会建立 AI 时代的"权威内容"

### 定位分析（找到你的切口）

**定位陈述模板：** 用这些框架来设计你的定位和市场沟通：

```
经典版：
针对 ___（目标用户），___（产品名）是一个 ___（品类），能 ___（核心价值），因为 ___（独特的理由）。

Before/After/Bridge（BAB）：
之前：___（痛苦的现状）
之后：___（理想状态）
桥梁：___（你的产品）如何帮用户从这里到那里

PAS（Problem-Agitate-Solution）：
问题：___（用户的痛苦）
放大：___（为什么现在不解决会越来越麻烦）
方案：___（你的方案为什么不同）
```

**定位地图：**
画出竞品在以下坐标系中的位置：

| 坐标轴 | X 轴左 | X 轴右 | Y 轴下 | Y 轴上 |
|---|---|---|---|---|
| 价值轴 | 低价 | 高价 | 基础功能 | 高级功能 |
| 用户轴 | 个人/小团队 | 企业 | 单点工具 | 平台 |
| 体验轴 | 复杂/高学习门槛 | 简单易用 | 功能少 | 功能全 |

> 白色空间 = 你的机会。如果所有竞品都在右上角（高价全功能），左下角（低价单点）就是你的切口。

**差异审计：** 问自己"Only-We 测试"——**只有你的产品能说 ___（因为独特的原因）**。如果你的竞品也能说同样的话，那就不是真正的差异。

### 痛点评分矩阵

用二维矩阵评估每个候选方向解决的痛点：

```
      严重性
      高 ┌─────────────────────────┐
         │   优先做（核心功能）    │
         │                         │
      中 │   做（辅助功能）        │
         │                         │
      低 │   暂不处理              │
         └─────────────────────────┘
         低       中       高   频率
```

- **高严重性 × 高频率：** 核心卖点，必须解决
- **高严重性 × 低频率：** 可以做，但不值得作为主要卖点
- **低严重性 × 高频率：** 辅助体验，不是购买决策因素
- **低严重性 × 低频率：** 放弃，或者等用户要求再做

### 市场拥挤度

**三层法：**
1. **竞品数量 × 质量**：多而强（难闯）/ 多而散（有切口）/ 少而冷（可能无需求）
2. **用户还在抱怨吗？** 持续抱怨 = 仍有缺口。评论挖掘里重复出现的差评是黄金信号
3. **找到六种切口之一：**

| 切口类型 | 问 | 示例 |
|---|---|---|
| 人群切口 | 有没有被忽视的用户群体？ | "为自由职业者做的项目工具" |
| 场景切口 | 有没有特定场景没被覆盖？ | "为移动端操作做的CRM" |
| 价格切口 | 能不能做一个比最便宜的竞品还便宜的版本？ | "$9/month vs $49/month" |
| 复杂度切口 | 能不能做一个只解决一件事的产品？ | "不做全栈分析，只做仪表盘截图" |
| 平台切口 | 有没有平台是竞品不支持的？ | "Linux版/微信小程序版" |
| 本地化切口 | 有没有语言/地区被忽视？ | "日语版/巴西版" |

> 拥挤判断必须下钻到人群和任务，不能停留在大类目。

### 付费意愿筛选

**弱信号（不可靠）：** "这个功能挺好" / "上线了告诉我" / "如果免费我会用"
**强信号（可靠）：** 已经花钱解决 / 已经花时间绕过 / 愿意留邮箱预约付钱

**付费公式：**
```
付费可能性 = 问题代价 × 发生频率 × 现有付费行为 × 切换成本可控
```
各 1-5 分，乘积越高越优先。

**四类假付费用户（避开）：** 只追新鲜感 / 无预算来源 / 问题太大你解决不了 / 只找免费替代

## 输出交付物

完成流程后，输出以下内容。

### 最终决策卡

```
我要服务 ___（目标用户）
在 ___（具体场景）里
解决 ___（具体问题）
因为现有竞品 ___（缺口）
我打算用 ___（更轻的方式）切进去
验证动作：___（7 天内可完成的验证）
```

### 附加材料
- 需求信号报告（至少 15-20 条原始信号，标注来源：Reddit/PH/GT/评论挖掘）
- 路径说明（走的是 A/B/C 哪条路径，为什么）
- 漏斗筛选记录（排除理由 + 五维打分表）
- 纵深分析表（竞品拆解 + SEO/GEO 扫描 + 定位分析 + 拥挤度评判 + 付费意愿评估）

## 工具箱

### 验证自查清单

完成分析后逐项检查：

- [ ] 信号来源 ≥ 2 个（不依赖单一来源做判断）
- [ ] 找到至少 10 条真实用户抱怨/需求（非产品经理脑补）
- [ ] 有竞品存在（完全没人做的方向 = 可能没人需要）
- [ ] 竞品有缺口（有人做但大家不满意）
- [ ] 目标人群能精准描述（不是"所有人"）
- [ ] 7 天内能完成一个验证动作
- [ ] 知道怎么收费（即使只收 1 块钱也要有定价）
- [ ] 决策卡写满，没有空项

## 工作示例（供参考）

以下是一个完整输出的示例，展示最终交付物长什么样：


## 规则

- **先判断用户输入类型**（路径 A/B/C），不要默认走同一条路
- **用户诊断时如果用户没回答全部问题，基于合理默认假设继续，但必须明确标注'以下是我的假设，请确认'**
- **路径 B（单方向）不走五层漏斗，直接用纵深分析 Phase 3。路径 C（多候选）先做真需求评估，再用五层漏斗筛**
- 每个阶段的产出必须有据可查（附来源或搜索过程）
- 不擅自说"做这个方向一定赚钱"——只给出验证假设
- 最终结论必须是**可验证的切口陈述**，不是宏大方向
- 所有验证动作控制在 7 天内完成
- 能 7 天验证的，不推荐花三个月开发
