# indie-product-direction

从"我想做什么"转向"市场已经证明了什么"。你不是在找灵感，你是在筛假设。

---

## 路径选择

用户输入分三种情况，先判断再选择路径：

```
"不知道做什么，帮我找方向"
  └→ 路径 A：用户诊断 → 信号发现

"我有个方向，值不值得做"
  └→ 路径 B：真需求评估 → 纵深分析（Phase 3）

"有好几个想法，帮我选一个"
  └→ 路径 C：真需求评估 → 五层漏斗
```

> 用户没说清楚时先问清楚。

---

## 路径 A：用户诊断 + 四大信号源

### 用户诊断

只用于"不知道做什么"的用户。先问 4 个问题：

1. **技能栈？** — 前端/后端/AI/全栈？
2. **领域知识？** — 跨境/教育/医疗/金融？
3. **投入时间？** — 兼职（~10h/周）/全职？
4. **收费模式？** — 订阅/一次性付费/免费？

> 如果用户没回答全部，基于合理默认假设继续，但标注"以下是我的假设，请确认"。

### 四大信号源

#### A：Reddit — 搜真实抱怨
- 定位 subreddit（AI→r/AI, SaaS→r/SaaS, SEO→r/SEO, App→r/AppIdeas）
- 搜痛点词：`looking for` / `alternative to` / `too expensive` / `hate this` / `manual work`
- 需求翻译法 —— 抱怨直接翻译成产品定义：

| 用户说 | 翻译后 |
|---|---|
| "截图调整尺寸太烦" | AI Screenshot Generator |
| "SEO 内容生成太慢" | AI SEO Automation Tool |

#### B：Product Hunt — 看已验证产品
- 研究排名 **10～50 名的中部产品**（非榜首）
- **先看定价页**（暴露商业逻辑）
- 记录：免费版限制、最低付费、高级版、企业版

#### C：Google Trends — 萌芽需求
- **看斜率不看绝对值** — 上涨小市场（+500%）比稳定大市场（+5%）更值钱
- 搜长尾词，关注 Related Queries 的 Breakout 爆发词

#### D：评论挖掘 — 差评里的需求
- 平台：SaaS→G2/Capterra, App→App Store/Google Play, 电商→Amazon
- 收集低分评论（2-3 星最有价值）；关注模式而非个案
- 信号 → 切口映射见下表：

| 评论信号 | 机会 |
|---|---|
| "太贵" | 价格切口 — 做轻量版 |
| "功能太多" | 复杂度切口 — 做单点工具 |
| "不适合我们" | 场景切口 — 做垂直版 |
| "不支持XX" | 平台/本地化切口 |

**目标产出：** 15-20 条信号 → 5 个候选方向 → 进入路径 C 的五层漏斗

## 路径 B：单方向评估（已有方向时）

不走五层漏斗，直接做真需求评估 + 纵深分析。

### 真需求评估

**公式：** `痛点（1-5） × 高频（1-5） × 付费（1-5）`

| 维度 | 1 分 | 5 分 |
|---|---|---|
| 痛点 | 不解决也能凑合 | 不解决就损失钱/客户/时间 |
| 高频 | 一年一次或更少 | 每天/每周 |
| 付费 | 全是免费工具 | 已有付费产品 |

> 总分 ≥8 → 深入 | 4-7 → 谨慎，先低成本验证 | <4 → 放弃

### 纵深分析

结果进入 Phase 3 做竞品分析、拥挤度、付费意愿评估。

---

## 路径 C：多方向筛选（五层漏斗）

先对每个方向做真需求评估，排掉明显不行的，剩下的走漏斗。

### 第一层：快速排除（三个"否"即排除）
1. 问题具体？2. 用户能感知？3. 现有方案麻烦？4. 用户可能付费？5. 能做小版本？

### 第二层：按用户问题分类
```
服务谁 → 解决什么 → 这个问题有什么代价
```

### 第三层：重复信号 — 五次以上同一问题反复出现→候选方向

### 第四层：五维打分（各 1-5）

| 维度 | 1 分 | 5 分 |
|---|---|---|
| 痛点强度 | 可有可无 | 不解决就难受 |
| 付费意愿 | 免费用 | 已在花钱 |
| 获客清晰度 | 一片模糊 | 知道去哪找用户 |
| 实现成本 | 耗时半年 | 一周搞定 |
| 竞争压力 | 巨头林立 | 没人认真做 |

### 第五层：7 天验证
```
具体人群 + 具体场景 + 具体问题 + 7 天验证动作
```
验证方式：落地页+邮箱 / Notion 模板 / 冷邮件 / **先收费再人工交付**（最强信号）

---

## Phase 3：纵深分析（三种路径都用）

### 竞品分析

**三圈扫描：**

| 圈层 | 类型 | 机会判断 |
|---|---|---|
| 第一圈 | 直接竞品（同类产品） | 看他们哪里做不够 |
| 第二圈 | 间接竞品（ChatGPT/人工服务） | 看用户为什么选这些 |
| 第三圈 | **土办法**（Excel/手工/微信群） | 有人用土办法=需求真实但没做好产品 |

**三线拆解：** 获客线（流量来源）/ 转化线（访客变试用）/ 留存线（为什么持续用）

**SEO 竞品扫描（量化维度）：** 收集 DA/DR、月均流量、Top 10 关键词数、反链质量、主要流量来源。DA 高流量大但口碑差→有机会用更好体验切走；DA 低增长快→新方向验证中。

**GEO/AI 引用分析：** 竞品内容是否被 AI 回答引用？定义性文章、统计数据最易被引用。竞品不被引用→你有机会建立 AI 时代的权威内容。

**定位分析：** 写定位陈述（经典版/BAB/PAS），画定位地图找白色空间，用 Only-We 测试验证差异。详细模板见 `references/positioning.md`。

**痛点评分矩阵：**

```
高严重性×高频率 = 核心卖点，必须解决
高严重性×低频率 = 可以做，但不做主卖点
低严重性×高频率 = 辅助体验
低严重性×低频率 = 放弃
```

### 市场拥挤度

**三层法：**
1. 竞品数量×质量：多而强（难闯）/ 多而散（有切口）/ 少而冷（可能无需求）
2. 用户还在抱怨吗？评论重复差评是黄金信号
3. 找到六种切口之一：人群/场景/价格/复杂度/平台/本地化

### 付费意愿

**弱信号（不可靠）：** "功能挺好" / "上线告诉我" / "免费我会用"
**强信号（可靠）：** 已花钱解决 / 已花时间绕过 / 留邮箱预约付钱

**公式：** `问题代价 × 发生频率 × 现有付费行为 × 切换成本可控`（各 1-5，越高越优先）

**避开四类假用户：** 只追新鲜感 / 无预算来源 / 问题太大你解决不了 / 只找免费替代

## 输出交付物

### 最终决策卡

```
我要服务 ___（目标用户）
在 ___（具体场景）里
解决 ___（具体问题）
因为现有竞品 ___（缺口）
我打算用 ___（更轻的方式）切进去
验证动作：___（7 天内完成）
```

### 附加材料
- 需求信号报告（15-20 条原始信号，标注来源）
- 路径说明（A/B/C 及理由）
- 漏斗筛选记录（排除理由 + 五维打分表）
- 纵深分析表（竞品拆解 + SEO/GEO + 定位 + 拥挤度 + 付费意愿）

## 规则

- 先判断用户输入类型（路径 A/B/C），不要默认走同一条路
- 路径 B（单方向）不走五层漏斗，直接用 Phase 3
- 路径 C（多候选）先真需求评估，再用五层漏斗
- 每个产出必须有据可查
- 只说"验证假设"，不说"一定赚钱"
- 最终结论必须是**可验证的切口陈述**，不是宏大方向
- 所有验证动作控制在 7 天内
