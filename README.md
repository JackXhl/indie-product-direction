# indie-product-direction

[![skills.sh](https://skills.sh/b/JackXhl/indie-product-direction)](https://skills.sh/JackXhl/indie-product-direction)

独立开发方向决策技能包。

从"不知道做什么"到"有一个可验证的产品假设"，一套完整的决策流程：**需求验证 → 信号采集 → 方向筛选 → 纵深分析 → 7 天验证**。

---

## 安装

### Claude Code

```bash
/plugin add https://github.com/JackXhl/indie-product-direction
```

### skills.sh

```bash
npx skills add https://github.com/JackXhl/indie-product-direction
```

### 手动

将 `SKILL.md` 放入 `.claude/skills/` 或 AI 工具指定的 skills 目录即可。

## 快速概览

**核心理念：** 从"我想做什么"转向"市场已经证明了什么"。

**真需求公式：** 痛点 × 高频 × 付费

```
Phase 1 四大信号源
  ├── Reddit — 搜用户抱怨
  ├── Product Hunt — 看排名 10-50 的中部产品
  ├── Google Trends — 看长尾词斜率
  └── 评论挖掘 — G2/Capterra/App Store 差评中找切口

Phase 2 五层漏斗 → 1 个可验证的产品假设

Phase 3 纵深分析
  ├── 竞品分析（三圈扫描 + 三线拆解 + SEO/GEO + 定位框架）
  ├── 市场拥挤度（三层法 + 六种切口）
  └── 付费意愿（强弱信号 + 筛选公式）

references/
  ├── checklist.md — 验证自查清单
  ├── battlecard.md — 竞品 Battlecard 模板
  ├── review-mining.md — 评论挖掘输出模板
  └── positioning.md — 定位框架参考
```

---

## 核心公式

```
真需求 = 痛点 × 高频 × 付费
付费可能性 = 问题代价 × 发生频率 × 现有付费行为 × 切换成本可控
市场机会 = 竞品"多而散" + 用户仍在抱怨 + 能找到六种切口之一
```
